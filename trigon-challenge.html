Step 2 · Choose test depth

How far should this run go?

Light, Medium, and Heavy describe process depth. They do not rank the importance of an idea, and a Heavy test never guarantees a stronger conclusion.

△

Light Test

A clean first pass for clarity, overlap, obvious failure points, and readiness for deeper work.

What it includes
  • Public-record scan
  • Question and framework definition
  • Small scenario set
  • Ambiguity, null, and AI-addition notes
△△

Medium Test

A structured test with sources, controls, cross-checks, locked variables, and reusable result data.

What it includes
  • Archive scan and geometry preflight
  • Locked variables and conditions
  • Baseline, null, and negative controls
  • Downloadable data when rights allow
△△△

Heavy Test

A full research lane for preregistration, data lock, failure gates, robustness checks, and reproducibility.

What it includes
  • Preregistered hypothesis and failure criteria
  • Frozen data and analysis choices
  • Robustness, sensitivity, and provenance checks
  • Code, data, checksums, and Record-P when cleared
Step 3 · Run the challenge

Simple on the surface. Expandable when needed.

The participant runs the first pass. AI may help organize, question, and record the test, but it does not become the judge and it may not silently repair the framework.

1Bring the idea

Framework, model, diagram, theory, system, or focused question.

2Start clean

Use a fresh or temporary AI chat when separation from prior context matters.

3Scan the record

Check the public record before treating the work as new.

4Lock the setup

Define the lane, depth, variables, controls, and failure conditions.

5Run scenarios

Test the framework against cases, prompts, files, or situations.

6Record honestly

Keep success, ambiguity, nulls, failures, and AI-added meaning visible.

What the challenge is

The Trigon Challenge is a participant-led pressure test. Your framework remains yours. The record separates original material, participant decisions, source material, and AI-added scaffolding.

AI can assist, but it does not judge. The participant runs the first pass.
What counts as a useful result?

A useful result may reveal consistency, ambiguity, a weak spot, a better next version, a reason to rebuild, or a need for clearer definitions. Failure can be more informative than vague success.

Privacy and separation

Use a fresh chat, Temporary Chat, or Memory Off when the platform provides it. Do not upload private, sensitive, or unpublished material unless you understand the platform settings and release implications.

Step 4 · Use the testing prompt

Copy the setup into a fresh AI chat.

Your selected lane and depth are added to the top. The original testing rules remain intact.

TRIGON TEST SETUP
Lane: [SELECT A LANE]
Depth: [SELECT A TEST DEPTH]

You are helping me run a Trigon Challenge test.

You are helping me organize the test, record results, flag ambiguity, and check consistency.

You are not the judge.
You are not allowed to invent missing rules.
You are not allowed to silently improve my framework.
You are not allowed to hide failures.

My framework is:
[PASTE FRAMEWORK NAME]

My framework description is:
[PASTE DESCRIPTION]

My framework parts are:
[PASTE PARTS, DOMAINS, REGIONS, VARIABLES, OR LAYERS]

My test question is:
[PASTE TEST QUESTION]

The scenarios I want to test are:
[PASTE SCENARIOS]

Important rules:
1. Do not rename my framework parts unless I ask.
2. Do not invent meanings for unnamed parts.
3. Do not treat ambiguity as success.
4. Do not hide failures.
5. Do not claim this proves my framework.
6. Do not claim this proves UBT.
7. Record uncertainty honestly.
8. Ask only necessary clarifying questions.

For each scenario, create a result row with:
- scenario ID
- scenario summary
- activated framework parts
- overlap or interaction
- central or integrating region activation, if relevant
- confidence from 0 to 100
- ambiguity notes
- failure or null notes
- whether extra meaning had to be added

Before we begin, restate the task neutrally and confirm that you will preserve ambiguity, failure, and uncertainty.
